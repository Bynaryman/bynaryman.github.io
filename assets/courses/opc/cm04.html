Backup: Primitive data types: Size

Original slide 153

The exact data type size is NOT part of the C standard
int can be 16 or 32 bits...
The standard only says that the range of a larger type shall be at least as big as the smaller type.
char short long long long
char: The minimum supported data type
Compiler decides based on the target platform architecture! (though, there is a minimum allowed width)
sizeof() can give the actual size in bytes
To give a specific storage size use the YintX_t type
Y = u for unsigned
X = 8, 16, 32 etc for the number of bits to be used

Backup: Logical operators

Original slide 160

  • The operands are TRUE or FALSE

  • A = 1, B = 0

  • Example

  • (A && B) is false.

  • (A || B) is true.

  • !(A && B) is true.

Op.
&&
||
!
  • Description
  • Logical AND: If both the operands are non-zero, then the condition becomes true.
  • Does not evaluate 2nd operand, if the first is false
  • Logical OR: If any of the two operands is non-zero, then the condition becomes true.
  • Does not evaluate 2nd operand, if the first is true
  • Logical NOT: It is used to reverse the logical state of its operand. If a condition is true, then Logical NOT operator will make it false.

Original slide 161

  • The operands are integral (short, int, unsigned, etc)

  • Op. Description Example

  • Binary AND: Copies a bit to the result, if it exists in both operands.

  • Binary OR: Copies a bit if it exists in either operand.

  • Binary XOR: Copies the bit if it is set only in one operand but not both.

  • Binary Ones Complement: Has the effect of 'flipping' bits.

  • Binary Left Shift: The left operands value is moved left by the number of bits specified by the right operand.

  • Binary Right Shift: The left operands value is moved right by the number of bits specified by the right operand.

Backup: And, when we cast different representation ?

Original slide 170

  • Casting float to int: It changes representation!
  • float is represented in IEEE 754 single precision
  • int is usually 2's Complement
  • float f = 3.9 ;
  • int val = (int) f ;
  • Value gets truncated !
  • 3.9 is truncated to 3.0 and stored as 3 in 2's complement
  • Casting int to float: It changes representation !
  • int f = 3 ;
  • float val = (float) f ;
  • Value to convert to float (thus, 3 is converted to 3.0)
